redflag.utils.aggregate(arr, absolute=False, rank_input=False, rank_output=False, normalize_input=False, normalize_output=False)Â¶

Compute the Borda count ranking from an N x M matrix representing N sets of rankings (or scores) for M candidates. This function aggregates the scores for each candidate and optionally normalizes them so that they sum to 1. The absolute value of the scores is considered if â€˜absoluteâ€™ is set to True.

If you are providing rankings like [1 (best), 2, 3] and so on, then set rank=True. If you also set normalize_output to False, you will get Borda ranking scores.

If your score arrays contain negative numbers and you want a large negative number to be considered â€˜strongâ€™, then set normalize_input to True.

Parameters:

  • arr (array-like) â€“ An N x M matrix where N is the number of sets of rankings or scores, and M is the number of candidates. Each element represents the score of a candidate in a particular set.

  • absolute (bool, optional) â€“ If True, the absolute value of each score is considered. This is useful when a large negative number should be considered as a strong score. Defaults to False.

  • rank_input (bool, optional) â€“ If True, them the input is transformed input ranking (such as [4 (best), 2, 3, â€¦]). Defaults to False.

  • rank_output (bool, optional) â€“ If True, the output will be the rankings of the aggregated scores instead of the scores themselves. This converts the aggregated scores into a rank format (such as [3 (best), 1, 2, â€¦]). Defaults to False.

  • normalize_input (bool, optional) â€“ If True, each row of the input array will be normalized before aggregation. This is useful when the input array contains values in different ranges or units and should be normalized to a common scale. Defaults to False.

  • normalize_output (bool, optional) â€“ If True, the aggregated scores will be normalized so that they sum to 1. This is useful to understand the relative importance of each candidateâ€™s score. Defaults to False.

Returns:

An array of length M containing the aggregated (and

optionally normalized) scores for each of the M candidates.

Return type:

numpy.ndarray

Example: >>> scores = ([ â€¦ [ 1, 0.25, 0], â€¦ [ 4, 1.5, 0], â€¦ [ 1, -0.25, 0] â€¦ ]) >>> aggregate(scores, normalize_output=True) array([0.8, 0.2, 0. ]) >>> aggregate(scores, absolute=True, normalize_output=True) array([0.75, 0.25, 0. ]) >>> scores = ([ â€¦ [ 1, 0.25, 0], â€¦ [ 4, 1.5, 0], â€¦ [ 1, -0.25, 0] â€¦ ]) >>> aggregate(scores, rank_input=True, rank_output=True) array([2, 1, 0])

redflag.utils.stdev_to_proportion(threshold: float, d: float = 1, n: float = 1000000000.0) → floatÂ¶

Estimate the confidence level of the scaled standard deviational hyperellipsoid (SDHE). This is the proportion of points whose Mahalanobis distance is within threshold standard deviations, for the given number of dimensions d.

For example, 68.27% of samples lie within Â±1 stdev of the mean in the univariate normal distribution. For two dimensions, d = 2 and 39.35% of the samples are within Â±1 stdev of the mean.

This is an approximation good to about 6 significant figures (depending on N). It uses the beta distribution to model the true distribution; for more about this see the following paper: http://poseidon.csd.auth.gr/papers/PUBLISHED/JOURNAL/pdf/Ververidis08a.pdf

For a table of test cases see Table 1 in: https://doi.org/10.1371/journal.pone.0118537

Parameters:

  • threshold (float) â€“ The number of standard deviations (or â€˜magnification ratioâ€™).

  • d (float) â€“ The number of dimensions.

  • n (float) â€“ The number of instances; just needs to be large for a proportion with decent precision.

Returns:

float. The confidence level.

Example

>>> stdev_to_proportion(1)  # Exact result: 0.6826894921370859
0.6826894916531445
>>> stdev_to_proportion(3)  # Exact result: 0.9973002039367398
0.9973002039633309
>>> stdev_to_proportion(1, d=2)
0.39346933952920327
>>> stdev_to_proportion(5, d=10)
